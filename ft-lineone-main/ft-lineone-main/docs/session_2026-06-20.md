## Problemas resueltos

### 1. Conexion a Supabase (PostgreSQL)
- **Problema**: Firewall corporativo (INACAP) bloquea puerto 5432. `asyncpg` timeout.
- **Solucion**: Migrar de conexion directa (SQLAlchemy + asyncpg) a **REST API** (PostgREST) sobre HTTPS:443.
- **Archivos creados**: `supabase_rest_client.py`, reescrito `supabase_client.py`

### 2. SDK supabase-py incompatible
- **Problema**: `supabase-py` v2.10.0 rechaza keys formato `sb_secret_*`/`sb_publishable_*` con "Invalid API key".
- **Solucion**: Reemplazar por `httpx` directo en todas las llamadas a Supabase Auth y REST.
- **Archivos**: `supabase_client.py` (reescrito), `supabase_rest_client.py` (nuevo)

### 3. Motor + Beanie + PyMongo: guerra de versiones
- **Problema**: Breaking changes en cadena entre pymongo 4.11+, motor 3.4+, beanie 2.x.
- **Solucion**: Pin a versiones compatibles:

| Paquete | Version | Nota |
|---------|---------|------|
| motor | 3.3.2 | Ultima compatible con pymongo <4.11 |
| pymongo | 4.8.0 | Ultima con `_QUERY_OPTIONS` en cursor |
| beanie | 1.30.0 | Ultima que no requiere pymongo.asynchronous |

### 4. Modelos Beanie mal definidos
- **Problema**: `ColorInfo`, `ProductImage`, `ProductAttributes` heredaban de `Document` (como si fueran colecciones) pero son sub-documentos embebidos.
- **Solucion**: Cambiar herencia a `BaseModel` de pydantic.

### 5. ZaraScraper no instanciaba
- **Problema**: `_fetch_page` decorado como `@abstractmethod` pero con implementacion concreta. Ningun scraper lo sobreescribia.
- **Solucion**: Quitar `@abstractmethod` de `_fetch_page`; `scrape_category` sigue siendo abstracto.

### 6. Errores graceful en startup
- **Problema**: Si MongoDB no esta disponible, el server falla al iniciar.
- **Solucion**: Envolver conexiones en try/except en `main.py` (MongoDB, ScraperPipeline, AdvisorAgent).
